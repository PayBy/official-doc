# Merchant Transfer Notification

## Overview

This notification is triggered when a merchant successfully transfers funds to a recipient account. Once the recipient account receives the payment, PayBy sends a notification to the merchant. The merchant must process this notification and respond accordingly.

## Notification Parameters

| Field Name                  | Variable Name       | Required | Type         | Example Value | Description                                                                 |
| -------------------------- | ------------------- | -------- | ------------ | ------------- | --------------------------------------------------------------------------- |
| Payment to Account Order   | `payToBalanceOrder` | Yes      | `RefundOrder`| -             | Contains details of the order related to the payment made to the account.  |

> **Note**: If additional fields are present in the `RefundOrder` object based on code samples, they should be documented in the corresponding object definition.

## Merchant Response

After receiving the notification, the merchant must return the following JSON response to indicate successful receipt:

```json
{"response":"SUCCESS"}
```

Any other response indicates an error or abnormal condition.

## Sample: Notification Handling

```json
{
  "payToBalanceOrder": {
    "merchantOrderId": "202309010001",
    "platformOrderId": "20230901000100001",
    "payeeLoginName": "test@payby.com",
    "payeeUserId": "100001",
    "amount": "100.00",
    "currency": "AED",
    "status": "SUCCESS",
    "gmtCreate": "2023-09-01T10:00:00+04:00",
    "gmtFinish": "2023-09-01T10:01:00+04:00"
  }
}
```

### `payToBalanceOrder` Attributes


| Field Name         | Variable Name     | Type     | Example Value           | Description                                                  |
| ------------------ | ----------------- | -------- | ------------------------ | ------------------------------------------------------------ |
| Merchant Order ID  | `merchantOrderId` | String   | `202309010001`           | Unique identifier for the order assigned by the merchant.    |
| Platform Order ID  | `platformOrderId` | String   | `20230901000100001`      | Unique identifier for the order assigned by PayBy.           |
| Payee Login Name   | `payeeLoginName`  | String   | `test@payby.com`         | Login name of the recipient account.                         |
| Payee User ID      | `payeeUserId`     | String   | `100001`                 | Unique user ID of the recipient.                             |
| Amount             | `amount`          | String   | `100.00`                 | Amount transferred to the account.                           |
| Currency           | `currency`        | String   | `AED`                    | Currency of the transferred amount.                          |
| Status             | `status`          | String   | `SUCCESS`                | Status of the payment transaction.                           |
| Created Time       | `gmtCreate`       | DateTime | `2023-09-01T10:00:00+04:00` | Timestamp when the order was created.                    |
| Finished Time      | `gmtFinish`       | DateTime | `2023-09-01T10:01:00+04:00` | Timestamp when the payment was completed.                |
