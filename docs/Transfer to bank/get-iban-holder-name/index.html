Query IBAN Holder Name

Overview​

This API verifies whether a provided account holder's name matches the corresponding IBAN. It is typically used to validate recipient information during fund transfers.

API URL​

  • UAT (Testing): https://uat.test2pay.com/sgs/api/transfer/getIbanHolderName
  • Production: https://api.payby.com/sgs/api/transfer/getIbanHolderName

Request Parameters​

HTTP Headers​

Attributes

  • Content-Language String

    • The language in which the response message will be used, currently only English is supported.
    • Example value: en
    • Maximum length: 10

  • Content-Type String Required

    • The media type. Required for operations with a request body. The value is application/<format>, where format is json.
    • Example value: application/json

  • sign String Required

    Requests should be signed using private-key cryptography. This allows the payment gateway to verify that an incoming request is really from your application.

  • Partner-Id String Required

    • The merchant id of your account.
    • Example value: 200001200101
    • Maximum length: 12

HTTP Body​

  • requestTime Timestamp Required

    Request time of the order. If the request time is more than 15 minutes away from the current time, the request will be rejected. This parameter is used to prevent repeated requests for orders that should have been cancelled due to timeouts.

    Example value: 1581493898000

  • bizContent Object

    Attributes

    • Merchant Order Number (merchantOrderNo)

      • Required: Yes
      • Type: String (max 64 characters)
      • Example: Me23484
      • Description: Unique identifier for the merchant's order.

    • Beneficiary Name (holderName)

      • Required: Yes
      • Type: String (max 100 characters)
      • Example: Bruce Lee
      • Description: Name of the beneficiary. Transmitted securely (encrypted).

    • IBAN (iban)

      • Required: No
      • Type: String (max 34 characters)
      • Example: -
      • Description: International Bank Account Number. Transmitted securely (encrypted).

    • SWIFT Code (swiftCode)

      • Required: No
      • Type: String (max 11 characters)
      • Example: ARABAEADDER
      • Description: Improves accuracy if provided.

    • Beneficiary Address (beneficiaryAddress)

      • Required: No
      • Type: String (max 140 characters)
      • Example: Liwa Street off Khaliffa Street
      • Description: Transmitted securely.
        If the bank account is personal, this field is required.
        The combined length of holderName and beneficiaryAddress must not exceed 140 characters, or the transfer may fail.

    • Amount (amount)

      • Required: Yes
      • Type: Money
      • Example: 12.34
      • Description: Payment amount for the transaction.

    • Payment Memo (memo)

      • Required: Yes
      • Type: String (max 128 characters)
      • Example: Bonus
      • Description: Remarks or notes for the payment.

    • Notification URL (notifyUrl)

      • Required: No
      • Type: String (max 200 characters)
      • Example: -
      • Description: URL for receiving backend notifications.

    • Account Number (accountNo)

      • Required: No
      • Type: String (max 64 characters)
      • Example: 2000000000001
      • Description: Transmitted securely (available since v2.1).

    • Network Code (networkCode)

      • Required: No
      • Type: String (max 16 characters)
      • Example: LOCAL
      • Description: Defaults to LOCAL (available since v2.1).

    • Bank Name (bankName)

      • Required: No
      • Type: String (max 50 characters)
      • Example: (empty)
      • Description: Name of the bank.

    • Country Code (countryCode)

      • Required: No
      • Type: String (max 16 characters)
      • Example: AE
      • Description: Defaults to AE (available since v2.1).
    • City Code (cityCode)
      • Required: No
      • Type: String (max 16 characters)
      • Example: (empty)
      • Description: Available since v2.3.
    • Payout Currency (fundoutCurrencyCode)
      • Required: No
      • Type: String (max 16 characters)
      • Example: AED
      • Description: Defaults to AED (available since v2.1).
    • Fedwire Code (fedwireCode)
      • Required: No
      • Type: String (max 9 characters)
      • Example: -
      • Description: Fedwire code (available since v2.1).
    • Branch Name (branchName)
      • Required: No
      • Type: String (max 35 characters)
      • Example: -
      • Description: Name of the bank branch.

    • Intermediary Bank (intermediaryBank)

      • Required: No
      • Type: String (max 11 characters)
      • Example: -
      • Description: Intermediary bank code (available since v2.1).

    • Beneficiary Type (beneficiaryType)

      • Required: No
      • Type: String (max 16 characters)
      • Example: IBAN
      • Description: Defaults to IBAN (available since v2.1).

    • Purpose Code (purposeCode)

      • Required: No
      • Type: String (max 16 characters)
      • Example: COM
      • Description: Code indicating the purpose of the transaction.

GetIbanHolderNameRequest​

  • Holder Name

    • Parameter Name: holderName
    • Required: Yes
    • Type: String(256)
    • Example: (none provided)
    • Description: Encrypted account holder name.

  • IBAN

    • Parameter Name: iban
    • Required: Yes
    • Type: String(256)
    • Example: (none provided)
    • Description: Encrypted IBAN number.
