Retrieve Order Details

Retrieves the order details with order number or merchant order number.


API URL​

UAT : https://uat.test2pay.com/sgs/api/transfer/getTransferToBankOrder

Production : https://api.payby.com/sgs/api/transfer/getTransferToBankOrder


Request​

Http Header​

Attributes

Content-Language String

The language in which the response message will be used, currently only English is supported.

Example value: en

Maximum length: 10


Content-Type String Required

The media type. Required for operations with a request body. The value is application/<format>, where format is json.

Example value: application/json


sign String Required

Requests should be signed using private-key cryptography. This allows the payment gateway to verify that an incoming request is really from your application.


Partner-Id String Required

The merchant id of your account.

Example value: 200001200101

Maximum length: 12


Http Body​

requestTime Timestamp Required

If the request time is more than 15 minutes away from the current time, the request will be rejected. This parameter is used to prevent repeated requests for orders that should have been cancelled due to timeouts.

Example value: 1581493898000


bizContent Object

Use merchantOrderNo to specify an order.

  • merchantOrderNo String Required

    The merchant's reference number of a specific transfer order..

    Example value: Me23484

    Maximum length: 64

Response​

Http Header​

sign String Required

When PayBy sends response, PayBy will use its own private key to sign the message, and the merchant uses PayBy 's public key to verify the signature. If the verification is passed, it proves that the response was sent by PayBy and not faked by others.


Http Body​

head

Attributes

  • applyStatus Enum Required

    The result of the request. The possible values are:

    • SUCCESS - Application successful.
    • FAIL - Application failed. Check the code and msg for exact reason.
    • ERROR - Application error. The signature verification failed. Please check whether the private key used for the signature and the public key uploaded on the PayBy portal are one key pair.

  • code String Required

    Response Codes.

    Maximum length: 10

    Example value: 0


  • msg String

    Description of this code.

    Maximum length: 200


  • traceCode String

    Internal code for locating the error.



body

Notice

Body is returned only when applystatus = success, and code = 0. If applystatus = error or failed; or applystatus = success, code !=0 , that indicates an error. Please check errors and try again.

  • transferToBankOrder Object

    Attributes
    • Request Time (requestTime)
      • Required: Yes
      • Type: Timestamp (milliseconds, 3-digit precision)
      • Example: 1581493898000
      • Description: Time the request was initiated.
    • Merchant Order Number (merchantOrderNo)
      • Required: Yes
      • Type: String (max 64 characters)
      • Example: S10000
      • Description: Unique identifier for the merchant's order.
    • PayBy Order Number (orderNo)
      • Required: Yes
      • Type: String (max 32 characters)
      • Example: O1000
      • Description: Unique identifier for the PayBy transaction.
    • Product Name (product)
      • Required: Yes
      • Type: String (max 200 characters)
      • Example: Transfer To Bank
      • Description: Name of the product or service.
    • Order Status (status)
      • Required: Yes
      • Type: (unspecified)
      • Example: -
      • Description: Current status of the order.
    • Payment Information (paymentInfo)

      • Required: No

      • Type: TransferToBankPaymentInfo object:

        • Payer Fee Amount

          • Variable Name: payerFeeAmount
          • Required: No
          • Type: Money
          • Example: -
          • Description: The fee charged to the sender (payer), including amount and currency.

        • Estimated Arrival Time

          • Variable Name: arrivalTime
          • Required: No
          • Type: Timestamp(3)
          • Example: 1585133416000
          • Description: Timestamp when the funds were expected to arrive or did arrive.

      • Example: 

        "paymentInfo": {
            "arrivalTime": 1585133416000,
            "payerFeeAmount": {
                "amount": 0,
                "currency": "AED"
            }
        }

      • Description: Detailed information about the payment processing, especially related to fees and timing.

    • Amount (amount)
      • Required: Yes
      • Type: Money
      • Example: (empty)
      • Description: Transaction amount.
    • Beneficiary Name (holderName)
      • Required: Yes
      • Type: String (max 64 characters)
      • Example: (empty)
      • Description: SHA-256 hash of the original name.
    • IBAN (Iban)
      • Required: Yes
      • Type: String (max 64 characters)
      • Example: (empty)
      • Description: SHA-256 hash of the original IBAN.
    • SWIFT Code (swiftCode)
      • Required: No
      • Type: String (max 11 characters)
      • Example: (empty)
      • Description: SWIFT code of the beneficiary bank.
    • Payment Memo (memo)
      • Required: No
      • Type: String (max 128 characters)
      • Example: (empty)
      • Description: Remarks or notes for the payment.
    • Notification URL (notifyUrl)
      • Required: No
      • Type: String (max 200 characters)
      • Example: (empty)
      • Description: URL for backend notifications.
    • Failure Description (failDes)
      • Required: No
      • Type: String (max 200 characters)
      • Example: -
      • Description: Reason for order failure.
    • Beneficiary Address (beneficiaryAddress)
      • Required: No
      • Type: String (max 200 characters)
      • Example: (empty)
      • Description: SHA-256 hash of the original address.
    • Bank Reference (bankReference)
      • Required: No
      • Type: String (max 128 characters)
      • Example: (empty)
      • Description: Reference number associated with the bank.
    • Payout Amount (fundoutAmount)
      • Required: No
      • Type: Money
      • Example: (empty)
      • Description: Amount credited to the beneficiary.
    • Exchange Rate (rate)
      • Required: No
      • Type: BigDecimal
      • Example: (empty)
      • Description: Exchange rate used for the transaction.
    • Network Code (networkCode)
      • Required: Yes
      • Type: String (max 20 characters)
      • Example: LOCAL
      • Description: Financial network used.
    • Country Code (countryCode)
      • Required: No
      • Type: String (max 16 characters)
      • Example: -
      • Description: Country code of the beneficiary bank.
    • Bank Name (bankName)
      • Required: No
      • Type: String (max 50 characters)
      • Example: -
      • Description: Name of the beneficiary bank.

    • Fedwire Code (fedwireCode)

      • Required: No
      • Type: String (max 9 characters)
      • Example: -
      • Description: Fedwire account code.

    • Intermediary Bank (intermediaryBank)

      • Required: No
      • Type: String (max 11 characters)
      • Example: -
      • Description: SWIFT code of the intermediary bank.
    • Branch Name (branchName)
      • Required: No
      • Type: String (max 35 characters)

      • Description: Name of the bank branch.
    • Beneficiary Type (beneficiaryType)
      • Required: Yes
      • Type: String (max 16 characters)

      • Description: Type of beneficiary account.
    • Purpose Code (purposeCode)
      • Required: No
      • Type: String (max 16 characters)
      • Example: COM
      • Description: Code indicating the purpose of the transaction.
