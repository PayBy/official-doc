Asynchronous Notification

Once a transfer to a bank account is processed, PayBy will send the result to the merchant. The merchant must handle the notification and respond accordingly. The actual time the funds arrive depends on the settlement times of the receiving bank.

Note

  1. Duplicate Notifications: The same notification may be sent multiple times. Your system must be able to handle duplicate notifications correctly.

  2. Retry Mechanism: If PayBy does not receive a valid or timely response from your system, it will consider the notification failed and retry sending it. PayBy will retry sending the notification up to 7 times at intervals of 2, 10, 10, 60, 120, 360, and 900 minutes, but final delivery success is not guaranteed.

  3. Uncertain Order Status: If the order status is unclear or no notification is received, merchants should actively query the transfer status via the API.

  4. Signature Verification: Notification requests are signed using PayBy’s RSA private key. Merchants must verify the signature using the PayBy public key, which can be downloaded from the merchant portal. The signing method is the same as for regular API requests.

Request​

Http Header​

Attributes

  • Content-Type String Required

    The media type. Required for operations with a request body. The value is application/<format>, where format is json.

    Example value: application/json

  • sign String Required

    When PayBy sends response, PayBy will use its own private key to sign the message, and the merchant uses PayBy 's public key to verify the signature. If the verification is passed, it proves that the response was sent by PayBy and not faked by others.

Http Body​

  • notify_timestamp Timestamp Required

    The timestamp when PayBy sent the request.

    Example value: 1586849271877

  • notify_id String Required

    The unique identification number of this notification within the PayBy system.

    Example value: 202004140007474501

  • transferToBankOrder Object

    Attributes

    • Request Time (requestTime)

      • Required: Yes
      • Type: Timestamp (milliseconds, 3-digit precision)
      • Example: 1581493898000
      • Description: Time the request was initiated.

    • Merchant Order Number (merchantOrderNo)

      • Required: Yes
      • Type: String (max 64 characters)
      • Example: S10000
      • Description: Unique identifier for the merchant's order.

    • PayBy Order Number (orderNo)

      • Required: Yes
      • Type: String (max 32 characters)
      • Example: O1000
      • Description: Unique identifier for the PayBy transaction.

    • Product Name (product)

      • Required: Yes
      • Type: String (max 200 characters)
      • Example: Transfer To Bank
      • Description: Name of the product or service.

    • Order Status (status)

      • Required: Yes
      • Type: (unspecified)
      • Example: -
      • Description: Current status of the order.

    • Payment Information (paymentInfo)

      • Required: No

      • Type: TransferToBankPaymentInfo object:

        • Payer Fee Amount

          • Variable Name: payerFeeAmount
          • Required: No
          • Type: Money
          • Example: -
          • Description: The fee charged to the sender (payer), including amount and currency.

        • Estimated Arrival Time

          • Variable Name: arrivalTime
          • Required: No
          • Type: Timestamp(3)
          • Example: 1585133416000
          • Description: Timestamp when the funds were expected to arrive or did arrive.

      • Example: 

        "paymentInfo": {
            "arrivalTime": 1585133416000,
            "payerFeeAmount": {
                "amount": 0,
                "currency": "AED"
            }
        }

      • Description: Detailed information about the payment processing, especially related to fees and timing.

    • Amount (amount)

      • Required: Yes
      • Type: Money
      • Example: (empty)
      • Description: Transaction amount.

    • Beneficiary Name (holderName)

      • Required: Yes
      • Type: String (max 64 characters)
      • Example: (empty)
      • Description: SHA-256 hash of the original name.

    • IBAN (Iban)

      • Required: Yes
      • Type: String (max 64 characters)
      • Example: (empty)
      • Description: SHA-256 hash of the original IBAN.

    • SWIFT Code (swiftCode)

      • Required: No
      • Type: String (max 11 characters)
      • Example: (empty)
      • Description: SWIFT code of the beneficiary bank.

    • Payment Memo (memo)

      • Required: No
      • Type: String (max 128 characters)
      • Example: (empty)
      • Description: Remarks or notes for the payment.

    • Notification URL (notifyUrl)

      • Required: No
      • Type: String (max 200 characters)
      • Example: (empty)
      • Description: URL for backend notifications.

    • Failure Description (failDes)

      • Required: No
      • Type: String (max 200 characters)
      • Example: -
      • Description: Reason for order failure.

    • Beneficiary Address (beneficiaryAddress)

      • Required: No
      • Type: String (max 200 characters)
      • Example: (empty)
      • Description: SHA-256 hash of the original address.

    • Bank Reference (bankReference)

      • Required: No
      • Type: String (max 128 characters)
      • Example: (empty)
      • Description: Reference number associated with the bank.

    • Payout Amount (fundoutAmount)

      • Required: No
      • Type: Money
      • Example: (empty)
      • Description: Amount credited to the beneficiary.

    • Exchange Rate (rate)

      • Required: No
      • Type: BigDecimal
      • Example: (empty)
      • Description: Exchange rate used for the transaction.

    • Network Code (networkCode)

      • Required: Yes
      • Type: String (max 20 characters)
      • Example: LOCAL
      • Description: Financial network used.

    • Country Code (countryCode)

      • Required: No
      • Type: String (max 16 characters)
      • Example: -
      • Description: Country code of the beneficiary bank.

    • Bank Name (bankName)

      • Required: No
      • Type: String (max 50 characters)
      • Example: -
      • Description: Name of the beneficiary bank.

    • Fedwire Code (fedwireCode)

      • Required: No
      • Type: String (max 9 characters)
      • Example: -
      • Description: Fedwire account code.

    • Intermediary Bank (intermediaryBank)

      • Required: No
      • Type: String (max 11 characters)
      • Example: -
      • Description: SWIFT code of the intermediary bank.

    • Branch Name (branchName)

      • Required: No
      • Type: String (max 35 characters)
      • Example: (empty)
      • Description: Name of the bank branch.

    • Beneficiary Type (beneficiaryType)

      • Required: Yes
      • Type: String (max 16 characters)
      • Example: IBAN
      • Description: Type of beneficiary account.

    • Purpose Code (purposeCode)

      • Required: No
      • Type: String (max 16 characters)
      • Example: COM
      • Description: Code indicating the purpose of the transaction.

      NameDescription
      COMCommission
      FISFinancial services
      GDSGoods Bought or Sold
      GMSProcessing repair and maintenance services on goods
      GOSGovernment goods and services embassies etc.
      GRIGovernment related income taxes, tariffs, capital transfers, etc.
      IFSInformation services
      IGTInter group transfer
      INSInsurance services
      ITSComputer services
      OATOwn account transfer
  • **notify_time** String

    The parameter is deprecated. Please use the notify_timestamp instead.

Response​

Please reply success after receiving the notification, otherwise PayBy will send the notification repeatedly for the same order.

Http Header​

  • Content-Type String Required

    The media type. Required for operations with a request body. The value is application/<format>, where format is json.

    Example value: application/json

Http Body​

  • response String Required

    Example value: Success

// Http Header
{
 "Content-Type": "application/json; charset=UTF-8" 
}

// Http Body
{
    "response":"SUCCESS"
}
